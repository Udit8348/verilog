# Verilog

## Simulation Infrastructure
To simulate my RTL, I initially evaluated two open-source options: **Icarus Verilog (iverilog)** and **Verilator**. I chose **Verilator** because it fits a workflow I like: keep the hardware in a *pure, synthesizable* HDL (`.v/.sv`) and drive it from a separate **C++ testbench**. That makes it easy to scale tests, reuse existing C++ utilities, and run fast simulations for larger designs.

Verilator works by compiling synthesizable RTL into C++ and then building an executable testbench. It also supports waveform dumping to **VCD**, which I use to inspect behavior in GTKWave.

### A “raw” Verilator testbench

A minimal Verilator testbench has a few recurring pieces:
- a DUT object (generated by Verilator, e.g. `Vtop`)
- optional trace support (`VerilatedVcdC`)
- a simulation timestamp used for waveform dumps (and, when needed, `sc_time_stamp()`)

See [`hello-verilator`](hello-verilator/) for full details.
```
#include "Vtop.h"
#include "verilated.h"
#include "verilated_vcd_c.h"

static vluint64_t main_time = 0;
double sc_time_stamp() { return static_cast<double>(main_time); }

static void step(Vtop* top, VerilatedVcdC* tfp, int cycles) {
    for (int i = 0; i < cycles; i++) {
        top->eval();
        if (tfp) tfp->dump(main_time);
        main_time++;
    }
}

int main(int argc, char** argv) {
    Verilated::commandArgs(argc, argv);

    Vtop* top = new Vtop;

    // Enable waveform dumping
    Verilated::traceEverOn(true);
    VerilatedVcdC* tfp = new VerilatedVcdC;

    top->trace(tfp, 5); // number of trace levels
    tfp->open("waveform.vcd");

    top->a = 0b00;
    top->b = 0b01;
    step(top, tfp, 2);

    top->final();
    tfp->close();
    delete tfp;
    delete top;
    return 0;
}
```

This works, but it repeats the same lifecycle and tracing boilerplate in every testbench.


Notice that we need to manage the lifetime of a few critical objects which are part of the Verilator framework: ` Vtop* top`, `VerilatedVcdC* tfp`, `vluint64_t main_time`.

To reduce repeated boilerplate, I wrapped the DUT + trace handling into a tiny Sim<TDut> helper class. It uses RAII to manage the DUT and VCD tracer lifetime and exposes a small, testbench-friendly API:
- `open_trace()` / `close_trace()` for VCD file handles
- `step()` for combinational evaluation and dumping
- `tick()` for clocked modules (assumes a clk signal named clk)
- `operator->` so the testbench can directly set DUT signals as `sim->signal = ...`

`Sim` full impl is found [`here`](lib/verilator_sim.h). It exposes the following interface:
```
inline vluint64_t g_time = 0;
inline double sc_time_stamp();

template <typename TDut>
class Sim {
public:
    Sim(int argc, char** argv);

    TDut* dut();

    TDut* operator->();
    const TDut* operator->() const;

    void open_trace(const char* vcd_path, int levels = 99);

    void close_trace();

    void step(int n = 1);

    void tick(int cycles = 1);

    ~Sim();

    Sim(const Sim&) = delete;
    Sim& operator=(const Sim&) = delete;
};
```

This is the earlier example rewritten with my wrapper:
```
#include "Vtop.h"
#include "lib/verilator_sim.h"

int main(int argc, char** argv) {
    Sim<Vtop> sim(argc, argv);
    sim.open_trace("waveform.vcd", 5);

    sim->a = 0b00;
    sim->b = 0b01;
    sim.step(2);

    // Optional: explicitly call final() on the underlying DUT
    sim.dut()->final();

    return 0;
}
```
Sim wraps a single Verilator-generated DUT and optional VCD tracing. For multi-module systems, I typically instantiate them under a single top-level RTL module (e.g., `top.sv`) and wrap that top in Sim. Most testbenches in this repo use this wrapper, however, if a testbench needs something custom, it can always drop down to the raw DUT pointer via `sim.dut()`.

I can use an LLM to quickly generate a `run.sh` script which does the following: discover HDL files, compile sim.cpp with Verilator, then use GTKWave to visualize the waveform.
